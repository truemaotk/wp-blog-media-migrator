# 博客文章与图片迁移工具

一个用于 WordPress 网站之间迁移博客文章的独立插件。它会迁移文章正文、分类标签、特色图、正文图片和常见阅读量字段，并提供状态选择、覆盖导入、进度提示及详细失败报告。

作者：[Mao TK](https://www.maotk.com/)

## 功能

- 分别选择已发布、私密、草稿、待审和定时文章。
- 显示每种文章状态的数量。
- 迁移标题、别名、完整正文、摘要和发布日期。
- 迁移分类、标签、文章密码及评论/引用状态。
- 迁移特色图片。
- 迁移正文中的普通图片、响应式图片、懒加载图片和 CSS 背景图。
- 迁移常见阅读量字段，以及字段名包含 `view`、`read` 或 `hit` 的数据。
- 可保持原文章状态，或统一导入为草稿。
- 可覆盖之前导入或别名相同的文章。
- 图片按内容哈希去重。
- 显示导出/导入进度、耗时和导出完成提示。
- 分别报告失败文章、失败图片和其他警告。

## 下载与安装

最新可安装包：

[`dist/wp-blog-media-migrator-1.2.1.zip`](dist/wp-blog-media-migrator-1.2.1.zip)

安装步骤：

1. 登录 WordPress 后台。
2. 打开“插件 → 安装插件 → 上传插件”。
3. 上传 ZIP 并安装。
4. 启用插件。
5. 打开“工具 → 博客文章与图片迁移”。

也可以把 `wp-blog-media-migrator` 文件夹上传到：

```text
wp-content/plugins/
```

## 完整迁移教程

### 一、选择文章状态

旧站导出页面会显示：

- 已发布：网站前台公开显示的文章
- 私密：仅登录且有权限的用户可查看
- 草稿：尚未发布
- 待审：等待审核
- 定时发布：计划未来自动发布

默认只勾选“已发布”，因此默认不会导出私密、草稿、待审或定时文章。

如果要迁移私密文章，请单独勾选“私密”。

### 二、在旧网站导出

1. 安装并启用插件。
2. 打开“工具 → 博客文章与图片迁移”。
3. 勾选需要导出的文章状态。
4. 点击“下载文章迁移包”。
5. 等待进度窗口显示“导出完成”。
6. 浏览器会下载包含文章清单和图片的 ZIP。

图片较多时，打包时间和 ZIP 体积都会增加。

### 三、备份新网站

导入前至少备份：

- WordPress 数据库
- `wp-content/uploads`
- 新站已有文章

### 四、在新网站导入

1. 在新站安装并启用同一插件。
2. 打开迁移工具。
3. 上传旧站生成的 ZIP。
4. 根据需要设置两个选项：
   - 覆盖已导入或别名相同的文章
   - 保留原文章状态
5. 点击“开始导入”。
6. 等待导入完成并查看报告。

## 两个导入选项

### 覆盖已导入或别名相同的文章

建议保持勾选。这样重复导入时会更新正文、分类、标签、特色图和阅读量，而不是继续创建重复文章。

插件首先使用“来源网站 + 原文章 ID”识别之前导入的文章；找不到时再按文章别名匹配。

### 保留原文章状态

勾选后：

- 已发布仍为已发布
- 私密仍为私密
- 草稿仍为草稿
- 待审仍为待审
- 定时文章仍保留定时状态

取消后，所有文章统一导入为草稿，适合先人工检查再发布。

## 阅读量迁移

不同主题和插件使用不同字段保存阅读量。本插件会迁移常见字段，例如：

```text
views
_views
post_views
post_views_count
view_count
views_count
zib_post_views
zibll_views
read_count
hits
```

也会识别字段名包含 `view`、`read`、`reading` 或 `hit` 的文章元数据。

注意：如果新旧网站使用完全不同的阅读量插件，数字虽然已经写入数据库，新主题仍可能读取另一个字段。此时需要根据新站实际字段做映射。

## 图片迁移

插件会尝试识别：

- `src`
- `srcset`
- `data-src`
- `data-original`
- `data-lazy-src`
- CSS `url(...)`
- 特色图片

导入成功后，会把正文中的旧图片地址替换为新站媒体库地址。

支持 PNG、JPEG、GIF、WebP、AVIF、BMP、ICO 和安全清理后的 SVG。

## 导入报告

结果页面会分别显示：

- 新增文章
- 覆盖文章
- 跳过文章
- 新增图片
- 失败文章：标题和具体原因
- 失败图片：文件名、旧站原地址和具体原因
- 其他警告：分类、标签等问题

如果全部成功，会明确显示成功提示。

## 服务器要求

- WordPress 5.8 或更高版本
- PHP 7.4 或更高版本
- PHP `ZipArchive` 扩展
- SVG 处理需要 PHP DOM 扩展
- 旧站能够访问文章内使用的图片

建议的大包配置：

```ini
memory_limit = 512M
max_execution_time = 600
upload_max_filesize = 1024M
post_max_size = 1024M
```

如果主机限制较低，可按文章状态分批导出，或者临时减少旧站文章数量。

## 注意事项

- 导入前务必备份。
- 私密文章可能含敏感内容，不要公开分享迁移 ZIP。
- 远程图片可能因防盗链、登录验证或链接失效而无法下载。
- 迁移不会复制原作者账号；文章将归当前执行导入的管理员或 WordPress 默认作者。
- 本插件不迁移评论、页面、商品、用户、主题设置或整个数据库。
- 定时文章依赖新站时区和 WordPress 定时任务，请导入后检查发布时间。
- 进度条表示服务器仍在处理，并非逐篇文章的精确百分比。
- 完成后抽查正文格式、图片、特色图、阅读量和私密权限。

## 常见问题

### 为什么导入包很大？

迁移包包含文章正文中的图片和特色图。图片数量及原图大小决定 ZIP 体积。

### 可以重复导入吗？

可以。建议勾选覆盖选项，插件会优先更新之前导入的文章并复用相同图片。

### 私密文章会被公开吗？

只有同时满足以下条件时才会保持私密：

1. 导出时勾选了“私密”。
2. 导入时勾选了“保留原文章状态”。

如果取消“保留原文章状态”，私密文章也会先保存为草稿，不会直接公开。

### 为什么阅读量没有显示？

通常是新主题读取的字段名与旧站不同。请查看导入报告并确认数据库已有阅读量字段，然后为新主题增加字段映射。
